# AzerothCore Project Service Boundary

- Project lifecycle: S0 — Proposed
- Scope: AzerothCore development project only

This boundary does not apply to or redefine blAIne as a whole.

The project may produce original tooling, modules, addons, control-plane software, documentation, configuration, presets, automation, integration work, training, and support services, subject to provenance and license review.

The project shall not provide hosted game realms, sell access to realms, or sell proprietary game content. Blizzard-owned client software, artwork, music, models, maps, data, and other proprietary assets shall not be project distribution artifacts.

Public or commercial release requires component-level provenance, licensing, attribution, sanitization, and distribution review. This is an engineering scope rule, not legal advice.
