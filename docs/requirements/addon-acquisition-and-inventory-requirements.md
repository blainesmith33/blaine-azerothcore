# Addon Acquisition and Inventory Requirements

- Project lifecycle: S0 — Proposed
- Current inventory status: NOT_STARTED

The user has a separate folder containing addons. Its contents have not been located, scanned, copied, renamed, hashed, modified, or classified by this operation.

A future, separately authorized inventory shall record human approval, exact source URL when known, author or maintainer, license, exact branch/tag/commit where applicable, acquisition timestamp, integrity hash where applicable, recommended and actually validated core baselines, dependencies, conflicts, server/client requirements, and disposition.

Assets may then be classified as:

- independent quality-of-life addon;
- upstream reference;
- modification candidate;
- companion candidate for a server module;
- replacement candidate for an original blAIne addon;
- obsolete;
- incompatible;
- duplicate;
- unresolved.

No classification is made now. Acquisition requires separate human approval and must preserve provenance and licensing.
